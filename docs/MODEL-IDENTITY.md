# 모델 아이덴티티 구상 — 고유 조작·기능·음색·시각 시그니처

> 2026-07-17 초안. 모든 기기(스킨)가 "같은 엔진의 다른 옷"에서 "다른 물건"이 되도록,
> 모델마다 ① 고유 조작 ② 고유 기능 ③ 음색 특성 ④ 시각 시그니처를 부여하기 위한 아이디어 모음.
> 구현 전 우선순위 논의용 — 확정 스펙 아님.

## 설계 원칙

1. **실물 근거주의** — 모든 고유 기능은 실제 그 모델(오마주 원기)의 대표 기능에서 가져온다.
   가짜 기능을 지어내지 않는다. (예: MR-78의 가변 선택도, B760의 프리셋, SL-1200의 피치)
2. **음색은 체인으로** — 앰프가 이미 가진 보이싱 테이블(drive/k/asym/톤/out) 패턴을
   튜너·데크·턴테이블로 확장한다. 각 기기는 작은 파라미터 객체 하나로 음색을 선언한다.
3. **WebKit 이중 대응** — 오디오 그래프는 SAFARI_LIKE에서 꺼져 있다.
   - *기계·시각 기능* (조작, 램프, 표시, 트랜스포트): 모든 브라우저에서 동작.
   - *DSP 음색* (보이싱, 필터, 배음): Chromium 계열에서만 실효, Safari에선 표시만.
   - *재생 속도* (피치·회전수): WebKit은 프레임 루프 내 playbackRate 대입 금지 —
     원샷 대입(applyRpmRate 패턴)으로는 가능. 피치 페이더는 드래그 종료 시 1회 대입.
4. **이미 그려진 것부터 살린다** — SL-1200 피치 페이더, TD 124 4속 노브, GARRARD
   SPEED·BRAKE 레버, DRAGON NAAC 패널, B215 12키 패널이 이미 SVG에 있으나 죽어 있다.
5. **조작 문법 유지** — 전면 정면도, 실물 조작(노브 드래그·스위치 클릭·레버), 과장 없는 조명.

## 튜너 (8)

공통 확장안: **수신 보이싱 테이블** `TUNER_VOICE[id] = { warmth, hissColor, blend, lockSpeed }`
— 같은 방송이라도 튜너마다 질감이 다르게. (DSP는 Chromium 한정, 램프·바늘 거동은 공통)

### YAHAMA T-2 — "서보 록의 정확함"
- 조작: 다이얼 릴리즈 시 **서보 오토터치** — 가까운 방송국으로 바늘이 스스로 미끄러져 안착 (T-2의 오토터치 튜닝).
- 기능: LOCKED LED가 안착 순간 한 번 깜빡이고 고정.
- 음색: 기준기 — 무색 (voice: 중립).
- 시각: 이미 확립된 그린 램프·오렌지 마크. 안착 시 마크가 짧게 밝아지는 "록 펄스".

### McIntoch MR-78 — "가변 선택도"
- 조작: 이미 그려진 **SELECTIVITY 스위치를 기능화** — WIDE/NORMAL/NARROW 3단.
- 기능: NARROW = 고역 감쇠 + 히스 제거(약전계 유리), WIDE = 개방적이나 히스 증가.
  이스터에그(105.2MHz)의 잡음 질감이 선택도에 반응하면 감동 포인트.
- 음색: 매킨토시식 "어두운 정숙" — NARROW 기본.
- 시각: 블루 글라스(확립). 선택도 단수마다 다이얼 창 밝기가 미세하게 변함.

### 10B — "오실로스코프의 눈"
- 조작: SCOPE 모드 스위치 — TUNING(멀티패스) ↔ AUDIO(리사주).
- 기능: 소형 **원형 스코프 CRT** — tsSignal/스테레오 위상으로 그린 리사주 애니메이션.
  튜닝 중엔 타원(멀티패스), 정확히 안착하면 곧은 선. *튜닝을 눈으로 하는 튜너.*
- 음색: 진공관 프런트엔드 — 미약한 2차 배음 온기 (voice.warmth 최고).
- 시각: 초록 인광 CRT — 잔상(persistence) 페이드. 이 모델만의 발광체.

### TU-9900 — "콰르츠 록"
- 조작: MUTING 스위치 기능화 — 선국 전환 시 무음 페이드(뚝 끊김 없음).
- 기능: T-LOCK LED — 안착 3초 후 점등되는 "쿼츠 잠금" 램프. 잠기면 바늘 미동이 멎는다.
- 음색: 아이스블루답게 차고 깨끗한 상단 (voice.blend 낮음).
- 시각: 아이스블루 램프(확립) + T-LOCK 청색 점등.

### PIONEER TX-9500 II — "약전계의 사냥꾼"
- 조작: MPX NOISE FILTER 스위치 기능화.
- 기능: 신호 약한 국(신호 시뮬레이션 값 낮음)에서 스테레오→모노 자동 블렌드 + 필터 램프.
- 음색: 필터 ON 시 고역이 살짝 접히는 대신 히스가 사라지는 거래.
- 시각: 은색+우드(확립). 신호 미터에 dBf 눈금 추가.

### LUXMAN T-110 — "출력을 가진 튜너"
- 조작: **OUTPUT LEVEL 노브** — 튜너 단의 송출 게인 트림 (앰프 볼륨과 별개).
- 기능: 라디오↔포노 전환 시 음량 차이를 이 노브로 맞추는 실사용 기능.
- 음색: 럭스만다운 부드러운 중역 (voice.warmth 중상).
- 시각: 노브 위치 백색 인디케이터 라인.

### ACCUPHASE T-100 — "계측기"
- 조작: 미터 모드 스위치 — SIGNAL ↔ MULTIPATH ↔ DEVIATION.
- 기능: 같은 바늘이 모드에 따라 다른 물리량을 보여줌 (멀티패스=지터 시뮬, 편차=변조도).
- 음색: 빠르고 개방적 (E-303과 동족 보이싱).
- 시각: 샴페인 패널(확립) + 미터 라벨이 모드 따라 교체 점등.

### REVOX B760 — "디지털 신시사이저" ★고유기능 대표
- 조작: **프리셋 키 1–8** — 길게 누르면 현재 국 저장, 짧게 누르면 호출. 키패드 주파수 직접 입력.
- 기능: 8국 메모리(localStorage) — 이 모델을 쓰는 이유가 되는 기능. 스캔 버튼(다음 국 자동).
- 음색: 신시사이저식 무결점 — 가장 무색, 히스 최소 (voice 중립+정숙).
- 시각: 그린 세그먼트 디지털 표시(확립) — 프리셋 호출 시 주파수가 카운트업되는 연출.

## EQ (4+1) — 밴드 구성은 이미 다름. 남은 건 성격 + 메모리.

- **GE-5 (WIDE RANGE)**: 조작감 그대로 — 5밴드 Q1.0 넓은 붓질. *기능*: 없음이 정체성 (가장 단순한 물건).
- **GE-10 (STUDIO MONITOR)**: *기능*: **분석기 스트립** — 슬라이더 위에 실시간 스펙트럼 LED 열 (이미 EQ LED 있음 → 입력 스펙트럼 표시 모드 추가). 스튜디오답게.
- **GE-10S (PRECISION)**: *조작*: **±6dB 정밀 레인지 스위치** — 같은 슬라이더 스트로크가 절반 게인. *시각*: 실버 헤어라인.
- **GE-10C (SIGNATURE)**: *음색*: **출력단 버퍼 새추레이션** — EQ를 통과하기만 해도 미세한 배음 윤기 (bypass와 A/B 되는 재미). *시각*: 샴페인+골드 캡.

### ★ SE-9 MEMORY (신설 제안) — "프리셋·메모리·A/B의 집" (2026-07-17, 사용자 제안 기반)

프리셋 뱅크를 전 모델 공용 UI로 띄우면 랙 문법이 깨진다. 대신 **메모리 EQ 모델을 신설**해
그 모델의 존재 이유로 삼는다. 실물 근거: Sansui SE-9 / Technics SH-8058 / ADC Sound Shaper
— 팩토리 커브 + 유저 메모리 + 커브 표시 + **모터 구동 슬라이더**가 실제로 있던 물건들.

**조작**
- 프리셋 키 2열 (팩토리 10 + 유저 A–D + MEMORY + A/B) — 각인 키캡, 선택 키 LED 점등.
- **MEMORY 쓰기**: MEMORY 키 점멸 상태에서 A–D 키 누르면 현재 커브 저장 (테이프 라벨 쓰듯).
- **A/B 레버 스위치**: 두 커브 뱅크 토글. *오토-B* — 어떤 프리셋이든 적용 직전 커브가
  자동으로 B에 들어간다 → 항상 "바꾸기 전/후"가 한 번의 스위치로 비교된다.
- DEFEAT는 기존대로 플랫 바이패스 (A/B와 별개의 3번째 비교축).

**연출 (킬러)** — 프리셋 호출 시 **슬라이더 캡이 목표 위치로 모터처럼 미끄러져 이동**
(300ms ease). 형광 커브 도트 디스플레이가 목표 커브를 먼저 그리고 캡이 따라간다.

**커브는 주파수-포인트로 정의, 밴드로 리샘플** — 프리셋을 (freq, dB) 포인트 배열로 선언하고
현재 모델의 밴드 주파수에 보간 적용. 같은 프리셋이 GE-5(5밴드)에서도 GE-10(10밴드)에서도
동작하고, 유저 슬롯 A–D도 모델을 갈아타도 유지된다. (SE-9에서 저장 → GE-5에서 호출 가능)

**팩토리 프리셋 10종** (10밴드 기준 dB, 31/62/125/250/500/1k/2k/4k/8k/16k · ±12 범위)

| 프리셋 | 커브 | 의도 |
|---|---|---|
| LATE NIGHT | +3/+2/+1/0/0/0/+0.5/+1/+1/0 | 저음량 등청감 보상 — 심야에 작게 틀어도 뼈대 유지 |
| VOCAL FOCUS | -2/-1.5/-1/-2/+1/+2.5/+3/+2/-0.5/-1 | 250Hz 탁함 컷 + 1–4k 존재감 |
| FM NOISE CUT | 0/0/+0.5/0/0/0/0/-1/-2.5/-5 | FM 히스(8k+) 정리, 몸통은 손대지 않음 |
| VINYL RESTORE | -4/-1.5/0/0/0/0/0/+1/+1.5/+1 | 럼블 컷 + 닳은 판의 광택 복원 |
| CASSETTE BRIGHT | 0/-1/0/0/0/0/+1/+2/+3.5/+4 | 테이프 고역 손실 보상 (헤드 범프 -1) |
| SMALL SPEAKER | -5/-3/+1.5/+2/+0.5/0/+0.5/+1/0/-0.5 | 못 내는 저음 컷 + 100–250 온기 착시 |

5밴드(60/250/1k/4k/12k)는 위 커브를 보간해 자동 산출 — 별도 표 불필요.

**맥락 점등 (선택)** — 지금 소스에 맞는 프리셋 키가 은은히 글로우: 라디오 수신 중엔
FM NOISE CUT, 포노 재생 중엔 VINYL RESTORE, 테이프 재생 중엔 CASSETTE BRIGHT.
"기계가 알고 있다"는 한 끗.

**구현 노트**
- 전환은 biquad `gain.setTargetAtTime`(~60ms)으로 크로스페이드 — 지퍼 노이즈 방지.
- 저장: fmRadio.eq에 `slots: {A,B,C,D}` (커브 포인트 배열) + `bankB` 추가.
- WebKit: 기존 EQ와 동일 — 슬라이더·표시·저장은 동작, 음향 반영은 Chromium 한정.
- LATE NIGHT의 등청감 보상은 CA-100 LOUDNESS(볼륨 연동)와 역할이 겹치지 않게 —
  EQ 프리셋은 고정 커브, LOUDNESS는 볼륨 추종이라는 구분 유지.

### Nakamichi DRAGON — "아무 테이프나 살려낸다"

- (대형 옵션) **오토 리버스** — 테이프 끝에서 B면으로 이어 재생. 테이프 모델에 면 개념 추가 필요 — 별도 설계 문서 필요.

- 시각: 그린 디지트(확립) + LOC 동작 중 카운터 깜빡임.

### TANDBERG TCD 3014A — "과대입력에 강하다"

### SONY TC-KA7ES — "정숙의 ES"
